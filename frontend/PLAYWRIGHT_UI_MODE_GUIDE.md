# Playwright UIモード 使い方ガイド

## UIモードとは

PlaywrightのUIモードは、テストを視覚的に確認しながら実行できるインタラクティブなモードです。

## 実行方法

```bash
npm run test:e2e:ui
```

## 使い方

### 1. UIが開いたら

ブラウザが自動的に開き、PlaywrightのUIが表示されます。

### 2. テストの選択

左側のパネルにテスト一覧が表示されます：
- テストファイル（例: `e2e/login.spec.ts`）
- テストケース（例: `ログインフォームが表示される`）

### 3. テストの実行

**方法1: 個別のテストを実行**
- 左側のパネルでテストをクリックして選択
- 右上の「▶ Run」ボタンをクリック

**方法2: 全てのテストを実行**
- 左上の「▶ Run all」ボタンをクリック

**方法3: ファイル全体を実行**
- テストファイル名をクリックして選択
- 「▶ Run」ボタンをクリック

### 4. 結果の確認

- テストの実行状況がリアルタイムで表示されます
- 成功したテストは緑色のチェックマーク
- 失敗したテストは赤色のXマーク
- スキップされたテストは灰色

### 5. 詳細の確認

テストをクリックすると、詳細情報が表示されます：
- 実行時間
- スクリーンショット（失敗時）
- エラーメッセージ（失敗時）

## トラブルシューティング

### UIが開かない、または中身が表示されない

1. **ブラウザが開いているか確認**
   - 新しいブラウザウィンドウが開いているはずです
   - タスクバーを確認してください

2. **URLを確認**
   - ブラウザのアドレスバーに `http://localhost:XXXX` が表示されているはずです

3. **通常モードで実行**
   - UIモードで問題がある場合は、通常モードを使用：
   ```bash
   npm run test:e2e
   ```

### テストが実行されない

- UIモードでは、**手動でテストを選択して実行ボタンを押す必要があります**
- 自動実行されません

## 通常モードとの違い

| 機能 | 通常モード | UIモード |
|------|-----------|---------|
| 実行方法 | 自動実行 | 手動選択 |
| 視覚的確認 | ターミナル出力 | ブラウザUI |
| デバッグ | 制限あり | 詳細な情報 |
| 実行速度 | 速い | やや遅い |

## 推奨される使用方法

- **開発中**: UIモードで個別のテストをデバッグ
- **CI/CD**: 通常モードで自動実行
- **最終確認**: 通常モードで全テスト実行
